# Review

## Summary
This paper introduces EXE, a benchmark for evaluating LLMs' code execution capabilities. It collects real-world code from GitHub to form a diverse and large dataset. The authors also propose some extensions to enhance the benchmark.

## Strengths
1. The benchmark is based on real-world data, enhancing its practical value and ecological validity.
2. The dataset is large in scale, comprising over 30,000 tasks, which allows for a more comprehensive evaluation of LLMs' capabilities.

## Weaknesses
1. The paper lacks a comparison with existing code execution benchmarks, such as CRUX-Eval, CodeX-Eval, and others. Including such a comparison would help position the proposed benchmark within the existing literature and highlight its unique contributions.
2. The paper does not provide a detailed analysis of how the proposed benchmark addresses the four challenges it mentions (scaling, task diversity, training data contamination, and cost-effective scalability). A more in-depth discussion of how the benchmark design mitigates or tackles these challenges would strengthen the paper.
3. The paper does not include an evaluation of open-source LLMs, such as Llama, which are known for their strong code generation capabilities. Including these models would provide a more comprehensive assessment of the benchmark's effectiveness across different LLMs.
